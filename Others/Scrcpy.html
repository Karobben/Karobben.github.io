How to install scrcpy
in Deepin 15.11


Karobben
5 Feb 2020



Scrcpy is a powful open source crassplatform software which could project your android phone to the computer. And it is Interoperable, it means you can control your phone with keyboard and mouse (You can input English only). Scrcpy is extremely easy to install at Ubuntu with the help of apt. But this way is unavallable in Deepin. However, you can still use "snap" to install it as a pice of cack in some regions. However, in other places like China, snap install are too slow to use. So, groups of people may inavitablly build scrcpy by source codes. It is easy for the people who are familiar with mason and nijia, but it is a nightmare for people like me, who find it is even hard to read the error codes. As a reuslt, I spent a few hours trying to figure out how to build this app in Deepin 15.11. So, I'd like to share the post and help the people like me to save the day.

I tried to install the latest version(1.12.1), but I have no idea why it asks me for the "scrcpy-server-v1.11.jar". So, I tried building V1.12.1 with scrcpy-server-v1.11.jar. It works in meson and nijia when you run conigure and build. When I run the scrcpy, everything just looks fine. Terminal resbonded well and my phone responded it, too. But there is no window jump out to the monitor. Creepy weird. So, I deleted all packages and installed the version V1.11.1, and boom, it finally works! Cheer!

1 Set up your environment


Terminal 

sudo apt install adb ffmpeg libsdl2-2.0.0 make gcc pkg-config meson ninja-build \
  libavcodec-dev libavformat-dev libavutil-dev libsdl2-dev

If the apt is running slow, please go switch to an appropriate source mirror.
An example of how to change your source:

Terminal 

## Generated by deepin-installer
deb [by-hash=force] http://ftp.sjtu.edu.cn/deepin panda main contrib non-free
#deb-src http://ftp.sjtu.edu.cn/deepin panda main contrib non-free




2 Dowloading Source Packages


Releas location: Genymobile Github Download the files in red boxs. In this section, we need to add ".jar" to the tail of scrcpy-server-v1.11 and install it later. And this is why I can't build it correctly at first two hours.

Terminal 

wget -c https://github.com/Genymobile/scrcpy/releases/download/v1.11/scrcpy-server-v1.11
wget -c https://github.com/Genymobile/scrcpy/archive/v1.11.tar.gz

mv scrcpy-server-v1.11 scrcpy-server-v1.11.jar
tar -zxvf v1.11.tar.gz




3 Install

Install the server

Terminal 

sudo install scrcpy-server-v1.11.jar /usr/local/bin/scrcpy-server.jar

Configure and Build

Terminal 

cd scrcpy
meson build --buildtype release --strip -Db_lto=true  -Dprebuilt_server=../scrcpy-server-v1.11.jar
cd build
ninja
sudo ninja install

You might receve this error

Terminal 

ninja: error: '/home/ken/Soft/scrcpy-server-v1.11.jar',
needed by 'server/scrcpy-server', missing and no known rule to make it

You can fix it by cp or move scrcpy-server-v1.11.jar to '/home/ken/Soft/scrcpy-server-v1.11.jar', in this example.




4 Project your Androird phone via USB

Connect your Androird Phone via USB, and open USB developer mode and make sure you enable USB debugging on your Android device.

Terminal 

wget -c https://github.com/Genymobile/scrcpy/releases/download/v1.11/scrcpy-server-v1.11
wget -c https://github.com/Genymobile/scrcpy/archive/v1.11.tar.gz

mv scrcpy-server-v1.11 scrcpy-server-v1.11.jar
tar -zxvf v1.11.tar.gz

Project your Androird phone without USB

For Wireless Connection, Following the instructor:

Terminal 

# Connect your phone via USB
adb tcpip 5555

# Now, disconnect the USB, and input:
adb connect 192.168.3.2:5555
scrcpy --bit-rate 4M

If rise error:

Terminal 

INFO: scrcpy 1.11 
[100%] /data/local/tmp/scrcpy-server
bind: Address already in use
ERROR: Could not listen on port 27183

Than:

Terminal 

scrcpy -p 1234

Arguments:


Terminal 

  --always-on-top
  Make scrcpy window always on top (above other windows).

-b, --bit-rate value
  Encode the video at the given bit-rate, expressed in bits/s.
  Unit suffixes are supported: 'K' (x1000) and 'M' (x1000000).
  Default is 8000000.

--crop width:height:x:y
  Crop the device screen on the server.
  The values are expressed in the device natural orientation
  (typically, portrait for a phone, landscape for a tablet).
  Any --max-size value is computed on the cropped size.

-f, --fullscreen
  Start in fullscreen.

-h, --help
  Print this help.

--max-fps value
  Limit the frame rate of screen capture (only supported on
  devices with Android >= 10).

-m, --max-size value
  Limit both the width and height of the video to value. The
  other dimension is computed so that the device aspect-ratio
  is preserved.
  Default is 0 (unlimited).

-n, --no-control
  Disable device control (mirror the device in read-only).

-N, --no-display
  Do not display device (only when screen recording is
  enabled).

-p, --port port
  Set the TCP port the client listens on.
  Default is 27183.

--prefer-text
  Inject alpha characters and space as text events instead of
  key events.
  This avoids issues when combining multiple keys to enter a
  special character, but breaks the expected behavior of alpha
  keys in games (typically WASD).

--push-target path
  Set the target directory for pushing files to the device by
  drag & drop. It is passed as-is to "adb push".
  Default is "/sdcard/".

-r, --record file.mp4
  Record screen to file.
  The format is determined by the --record-format option if
  set, or by the file extension (.mp4 or .mkv).

--record-format format
  Force recording format (either mp4 or mkv).

--render-expired-frames
  By default, to minimize latency, scrcpy always renders the
  last available decoded frame, and drops any previous ones.
  This flag forces to render all frames, at a cost of a

-s, --serial serial
  The device serial number. Mandatory only if several devices

-S, --turn-screen-off
  Turn the device screen off immediately.

-t, --show-touches
  Enable "show touches" on start, disable on quit.
  It only shows physical touches (not clicks from scrcpy).

-v, --version
  Print the version of scrcpy.

--window-borderless
  Disable window decorations (display borderless window).

--window-title text
  Set a custom window title.

--window-x value
  Set the initial window horizontal position.

--window-y value
  Set the initial window vertical position.
  Default is -1 (automatic).

--window-width value
  Set the initial window width.
  Default is 0 (automatic).

--window-height value
  Set the initial window width.
  Default is 0 (automatic).
